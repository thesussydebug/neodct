NeoDCT Changelog

Version 0.3.0a (in progress) - 2026-07-14

Modem (SIM7600G bring-up -- MODEM SUPPORT!)
- QEMU kernel config now ships the drivers the SIM7600 needs and they are
  persisted in board/qemu/aarch64-virt/linux.config (they used to live only
  in the ephemeral build .config): USB_SERIAL_OPTION (the driver that
  actually creates /dev/ttyUSB0-4 for 1e0e:9001 -- its absence was why
  lsusb saw the modem but no tty nodes appeared), plus QMI_WWAN for a
  wwan0 interface at the stock PID, ACM/usbnet/RNDIS carried over.
- New atcmd helper (engineering/tools/atcmd): one scriptable AT command
  per call over BusyBox sh, with an advisory lock shared with the boot
  script and the UI so replies never interleave.
- New /etc/init.d/S45modem: backgrounded boot bring-up -- SIM check, PDP
  context (IPV6 / fast.t-mobile.com by default, /etc/default/modem to
  override), LTE registration wait, AT$QCRMCALL data call, wwan0 + SLAAC
  + DNS64. Logs to /tmp/modem-boot.log, status in /tmp/modem.status.
- ModemService is REAL now: owns the AT port via raw termios (no pyserial,
  no ModemManager, no threads), auto-probes ports by USB interface number,
  polls AT+CSQ/+CEREG/+COPS on staggered timers from the main loop, parses
  call URCs (RING/+CLIP/VOICE CALL/NO CARRIER) into a state machine plus
  event queue, and dial()/hangup() send real ATD/AT+CHUP. Falls back to
  Simulation Mode like BatteryService (sim hooks: /tmp/neodct_sim_csq,
  /tmp/neodct_sim_ring) and hot-adopts a modem that appears later.
- Home screen signal bars are LIVE: the sig icon_set now renders 0-4 bars
  from real tower CSQ (0 bars when unregistered); sim_val still applies in
  Simulation Mode.
- docs/MODEM_BRINGUP.md: full runbook -- QEMU passthrough gotchas (host
  ModemManager, PID-switch warning), terminal proof-of-concept sequence,
  S45modem config, ModemService integration notes, troubleshooting table.

Version 0.2.4a - 2026-07-13

Power/Battery
- New BatteryService (System/core/BatteryService): reads VCELL from a
  MAX1704x fuel gauge over I2C (bus/addr configurable via
  system.hw.battery_i2c_bus / system.hw.battery_i2c_addr, default 1/0x36).
- Home screen battery gauge is now live: <3.35 V dead (bat-0), 0.20 V bands
  up to >=3.95 V full (bat-4). Readings are smoothed over a 5-sample window.
- LOW BATTERY! warning at <=3.45 V, BATTERY CRITICALLY LOW! at <=3.25 V
  (one-shot, re-armed if the pack recovers), graceful poweroff at <=3.20 V
  (confirmed over 3 consecutive polls; syncs filesystems first).
- QEMU/dev fallback: without gauge hardware the service runs in Simulation
  Mode (boot log says which mode is active, like the modem stub). Simulated
  voltage overridable via NEODCT_BATT_SIM_VCELL env var or by writing a
  value to /tmp/neodct_sim_vcell at runtime.
- Added engineering/tools/max1704x_watch.py bench utility used for the
  Raspberry Pi supply-sweep prototyping.

Engineering
- New FuelGauge engineering app (id 9004, diesel-can icon): live MAX1704x
  readout (VCELL/SOC/CRATE/VERSION/CONFIG + system gauge state) refreshed
  every second through the shared BatteryService handle. Softkey issues a
  gauge quick-start (re-seeds SOC from VCELL for bench sweeps). Shows a
  notice and exits when the gauge is simulated (QEMU / missing hardware),
  like the other hardware-bound engineering apps.
- BatteryService grew debug_snapshot() and quickstart() to back the app.
- Engineering apps finally lost the hard-hat placeholder icon: CubeBench
  (3D cube), KeypadMapper (access-keypad photo), KeypadMapperI2C (same
  photo + I2C badge), Crash (red X ball, redrawn crisp), LCDTest
  (monitor) and Tests (checklist clipboard). All 120x120 transparent
  PNGs; FuelGauge and LinuxShell keep their existing icons.

Framework
- MessageDialog.render() draws a dialog without blocking on a key;
  ErrorScreen.show_error(wait_for_ack=False) exposes it for the shutdown
  notice.
- MessageDialog layout fixes: body text no longer overlaps the warning
  icon (affected every dialog incl. the alpha notice), the body block is
  vertically centered, and short notices (<=2 lines) now render
  Nokia-alert style: normal font, centered. Paragraphs keep the small
  left-aligned layout.
- Home layout icon_set elements now honor sim_val instead of a hardcoded
  level (cell signal still simulated until the modem goes real).

Versioning
- Version bumped to 0.2.4a; stale version strings in an existing
  settings.prop are now refreshed to the installed build.

Version 0.2.0a - 2026-02-25

Summary
- Major milestone release focused on real hardware constraints and performance.
- Moved directly to 0.2.0a due to scope and platform changes.

Build Organization and Milestones
- 0.2 introduces a cleaner split between software versioning and hardware progress tracking.
- Build identifiers now stay software-only (for example: `v0.2.0a`), without milestone tags in the build name.
- Hardware stages (PROTO/EVT/DVT/PVT) are now treated as parallel project-tracking states, not version suffixes.
- This allows simultaneous work across tracks (for example, PROTO0 experiments and EVT0 target bring-up in the same timeframe) without version-label confusion.
- Legacy milestone-style build labels were dropped as part of this refactor.

Platform
- Primary SBC target changed from Radxa Zero 3W to Luckfox Pico Mini B.
- Luckfox profile now reflects practical embedded constraints for NeoDCT engineering:
  - CPU: single Cortex-A7 @ 1.2 GHz
  - RAM: 64 MB
  - Storage: built-in 128 MB SPI flash + SD expansion
- Buildroot is now running on real hardware for this track.

System/Buildroot
- Slimmed package set for core UI path and removed heavyweight desktop/browser stack from the lean configs:
  - removed/disabled: Cage, Wayland/Wlroots/Wlr-randr, X.org stack, WebKitGTK, NetSurf, Mesa stack, and related extras in lean profiles.
- Kept minimum runtime set for NeoDCT UI and lightweight browsing workflow:
  - Python3 (+ssl/sqlite/xz), Pillow, FreeType, DejaVu, CA certs, Links (+graphics), GPM.
- Re-added python-mutagen for Music Player metadata/art handling.
- Fixed overlay pathing in active configs to `../neodct/overlay`.
- Added split-output workflow support (`O=...`) for separate QEMU and Luckfox build trees.

Rendering/Performance
- Core framebuffer pipeline in `System/core/main.py` was optimized (no external renderer split):
  - reduced per-frame allocations by reusing buffers/images
  - reduced unnecessary conversions/crops
  - optimized RGB565 fallback path with lookup tables and reusable output buffers
  - added partial center-band write path for common UI case to avoid unnecessary full-frame work
- Result: noticeably smoother software rendering in QEMU bottleneck testing and improved perceived UI smoothness.

Engineering/Apps
- Engineering app layout standardized under `/NeoDCT/System/engineering`.
- Added CubeBench engineering app (pure software PIL 3D spinning cube) as a baseline rendering benchmark.
- Added Luckfox-specific init behavior:
  - `inittab.luckfox` with serial console handling for hardware bring-up
  - guarded post-build application so Luckfox-specific inittab does not leak into QEMU builds
  - serial redundancy configured for `ttyFIQ0` and `ttyAMA0` in Luckfox profile
- Luckfox post-build currently prunes `/tests` for smaller rootfs in that profile.

Versioning/UI
- Updated version strings/banners to 0.2.0a context.
- QEMU banner and settings fields were aligned during transition work.

Browser Direction
- WebKitGTK is no longer viable for the 64 MB RAM target.
- Browser app path has shifted to lightweight Links for now.
- Full web experience will require a reworked approach suitable for ultra-constrained memory.

Notes
- This release is primarily an under-the-hood engineering leap toward true device-level constraints, not just emulator convenience.
- The new constraints are intentionally closer to classic feature-phone-era engineering tradeoffs.
